# Conciencia y Salud - Campaña de Vacunación contra el Sarampión

## Problemática

En los últimos meses se ha detectado un nuevo brote de sarampión que representa un riesgo significativo para la comunidad universitaria. Muchos estudiantes desconocen si cuentan con su esquema de vacunación completo, no tienen acceso fácil a información médica confiable sobre la enfermedad y carecen de un medio accesible para agendar una cita de vacunación. La falta de concientización y de herramientas prácticas puede facilitar la propagación del virus dentro de los campus universitarios.

## Objetivo

Desarrollar un sitio web informativo y funcional que permita a los estudiantes universitarios conocer los riesgos del sarampión, acceder a información médica clara sobre la enfermedad y agendar una cita de vacunación de forma sencilla en alguna de las sedes disponibles. El sitio busca servir como herramienta principal de una campaña de concientización y vacunación dirigida a la comunidad universitaria.

## Conflicto generado
El conflicto ocurrió en el archivo index.html porque dos integrantes del equipo modificaron exactamente el mismo párrafo de la descripción principal en sus respectivas ramas de trabajo. Git detectó que el mismo bloque de código tenía dos versiones distintas y, al intentar realizar el merge hacia la rama main, no pudo determinar automáticamente cuál versión conservar.


## Cómo lo resolvimos
Para solucionar este inconveniente, seguimos el siguiente proceso colaborativo:


## Identificación: 
Localizamos las marcas de conflicto (<<<<<<< HEAD, =======, >>>>>>>) dentro del código.



## Análisis: 
Comparamos la versión del Integrante A (enfocada en mejorar la problemática) con la del Integrante B (enfocada en optimizar mediante una solución digital).


## Decisión: 
Optamos por crear una versión combinada que integra ambas propuestas.


## Limpieza y Commit: 
Eliminamos manualmente las marcas de Git, guardamos los cambios y ejecutamos los comandos git add . y git commit -m "Conflicto resuelto colaborativamente en descripcion principal" para cerrar el ciclo del conflicto.

## Qué aprendimos
A través de este ejercicio, el equipo obtuvo las siguientes reflexiones:


La comunicación es vital: Los conflictos no son errores del programador, sino situaciones normales del desarrollo colaborativo que requieren diálogo para elegir la mejor solución técnica.

Control de versiones: Entendimos cómo Git protege la integridad del código al detener un merge que podría causar pérdida de información.

Autonomía técnica: Ahora sabemos interpretar y limpiar las marcas de conflicto de manera manual, lo cual es una habilidad fundamental para nuestra carrera en Ingeniería en Tecnologías de la Información.
